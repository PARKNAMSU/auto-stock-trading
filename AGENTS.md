# 프로젝트 작업 지침

## 세션 시작

- 작업을 시작하기 전에 루트의 `HANDOFF.md`를 읽는다.
- `git status --short`와 최근 커밋을 확인해 현재 작업 상태를 파악한다.
- 사용자가 진행할 이슈를 지정하지 않았다면 GitHub의 열린 이슈와 `priority:P0` 라벨을 확인한다.
- 기존 사용자 변경사항과 관련 없는 파일은 수정하거나 되돌리지 않는다.

## 구현 및 검증

- Go 코드는 `gofmt`를 적용한다.
- 변경 후 최소한 `go test ./...`를 실행한다.
- 테스트는 루트의 `test/<package>` 디렉터리에 두고 패키지 이름은 `<package>_test`로 작성한다.
- 가능한 경우 테스트는 대상 패키지의 공개 API를 통해 작성한다.
- 토스증권 API 관련 작업은 먼저 `toss-doc/README.md`와 공식 문서 원본을 확인하고, 정확한 스키마는 `toss-doc/openapi.json`을 최종 기준으로 삼는다.
- 인증정보, 계좌정보 및 토큰을 코드, 로그, 문서에 기록하지 않는다.

## GitHub 및 Git

- 구현 작업은 관련 GitHub 이슈를 기준으로 진행하고 결과에 이슈 번호를 남긴다.
- 사용자가 요청하지 않은 커밋, 푸시, 이슈 종료 또는 PR 생성은 하지 않는다.
- 커밋하기 전에 변경 파일과 테스트 결과를 확인한다.

### 커밋 메시지 규칙

- 제목은 `[타입] 작업 제목` 형식으로 작성한다.
- 타입은 작업 성격에 따라 다음 중 하나를 사용한다.
  - `feat`: 새로운 기능을 추가하거나 기존 기능을 확장한다.
  - `fix`: 오류 또는 잘못된 동작을 수정한다.
  - `chore`: 설정, 문서, 테스트, 의존성 및 기타 유지보수 작업을 수행한다.
- 제목 다음에 빈 줄을 넣고, 본문에는 주요 작업 내용을 `- ` 목록으로 작성한다.
- 연결된 GitHub 이슈가 있으면 본문 마지막에 빈 줄을 넣고 `Issue: 이슈 URL`을 작성한다.
- 연결된 이슈가 없으면 `Issue:` 줄을 생략한다.

```text
[feat] 토스증권 API 인증 구현

- OAuth2 액세스 토큰 발급 기능 추가
- 토큰 만료 및 재발급 처리 추가

Issue: https://github.com/PARKNAMSU/auto-stock-trading/issues/2
```

이슈가 없는 경우에는 다음과 같이 작성한다.

```text
[chore] 프로젝트 설정 정리

- 불필요한 설정 제거
- 예제 환경변수 설명 보완
```

## 세션 종료

- 의미 있는 작업 상태가 바뀌면 `HANDOFF.md`의 완료 작업, 현재 상태, 다음 작업을 갱신한다.
- 미완료 작업과 알려진 문제를 숨기지 않고 명시한다.
